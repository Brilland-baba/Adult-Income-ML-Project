5. PR√âTRAITEMENT DES DONN√âES

üîß Strat√©gie de Transformation

Utilisation de ColumnTransformer de scikit-learn pour appliquer des transformations sp√©cifiques selon le type de variable.

Branch 1: Variables Asym√©triques

Transformation: log1p ‚Üí StandardScaler

Variables: capital_gain, capital_loss

Justification: La transformation log1p normalise les distributions fortement asym√©triques et g√®re les valeurs nulles (log(0+1) = 0).

Branch 2: Variables Normales

Transformation: StandardScaler uniquement

Variables: age, education_num, hours_per_week

Justification: Distribution d√©j√† relativement sym√©trique, seule la standardisation (moyenne=0, √©cart-type=1) est n√©cessaire.

Branch 3: Variables Cat√©gorielles

R√©sultat: G√©n√©ration d'environ 70 colonnes binaires (0/1)

from sklearn.preprocessing import FunctionTransformer, StandardScaler, OneHotEncoder from sklearn.compose import ColumnTransformer from sklearn.pipeline import make_pipeline from sklearn.impute import SimpleImputer # Identification des variables asym√©triques high_skew_features = ['capital_gain', 'capital_loss'] normal_numeric = ['age', 'education_num', 'hours_per_week'] cat_cols = ['workclass', 'marital_status', 'occupation', 'relationship', 'race', 'sex'] # Pipeline pour variables asym√©triques log_transformer = make_pipeline( SimpleImputer(strategy="median"), FunctionTransformer(np.log1p, feature_names_out="one-to-one"), StandardScaler() ) # Pipeline pour variables normales numeric_transformer = make_pipeline( SimpleImputer(strategy="median"), StandardScaler() ) # Pipeline pour variables cat√©gorielles categorical_transformer = make_pipeline( SimpleImputer(strategy="constant", fill_value="missing"), OneHotEncoder(handle_unknown="ignore", sparse_output=False) ) # ColumnTransformer final preprocess = ColumnTransformer([ ("log", log_transformer, high_skew_features), ("num", numeric_transformer, normal_numeric), ("cat", categorical_transformer, cat_cols), ], remainder="drop")

‚öñÔ∏è R√©√©quilibrage avec SMOTE

üî¨ SMOTE = Synthetic Minority Over-sampling Technique

Technique d'oversampling intelligente qui cr√©e des exemples synth√©tiques de la classe minoritaire au lieu de simplement dupliquer les observations existantes.

üéØ Fonctionnement de SMOTE

  1. S√©lection: Pour chaque observation de la classe minoritaire, SMOTE s√©lectionne ses k plus proches voisins (k=5 par d√©faut)
  2. Interpolation: Un voisin est choisi al√©atoirement, puis un nouveau point est cr√©√© par interpolation lin√©aire entre l'observation et son voisin
  3. R√©p√©tition: Le processus est r√©p√©t√© jusqu'√† atteindre l'√©quilibre souhait√© (ratio 1:1)

‚ùå Avant SMOTE

3.2:1

Classe 0: 24,720

Classe 1: 7,841

‚úÖ Apr√®s SMOTE

1:1

Classe 0: 24,720

Classe 1: 24,720

üíª Impl√©mentation avec ImbPipeline

‚ö†Ô∏è IMPORTANT: Protection contre le Data Leakage

Utilisation de ImbPipeline (de imblearn) pour garantir que SMOTE est appliqu√© uniquement sur les donn√©es d'entra√Ænement de chaque fold, jamais sur les donn√©es de validation.

from imblearn.over_sampling import SMOTE from imblearn.pipeline import Pipeline as ImbPipeline # Pipeline complet avec SMOTE pipe = ImbPipeline([ ("preprocess", preprocess), # Transformation features ("smote", SMOTE(random_state=42)), # R√©√©quilibrage ("clf", classifier) # Mod√®le ML ]) # En validation crois√©e, SMOTE est appliqu√© dans chaque fold cv_scores = cross_val_score(pipe, X, y, cv=5, scoring="roc_auc")

üéØ Strat√©gie de Validation Crois√©e

üìä ROC AUC - La M√©trique Principale

Pourquoi ROC AUC plut√¥t qu'Accuracy?

Sur un dataset d√©s√©quilibr√© (76%-24%), un mod√®le qui pr√©dit toujours la classe majoritaire obtient 76% d'accuracy mais AUC = 0.5 (al√©atoire). ROC AUC mesure vraiment la capacit√© discriminante du mod√®le.

Score AUC Interpr√©tation Qualit√©
AUC = 0.5 Performance al√©atoire Mauvais
0.7 ‚â§ AUC < 0.8 Performance acceptable Moyen
0.8 ‚â§ AUC < 0.9 Excellente performance Bon
